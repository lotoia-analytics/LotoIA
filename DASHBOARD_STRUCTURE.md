# Dashboard Structure

## Overview

The institutional dashboard is implemented in `dashboard/admin_app.py`.
It provides the primary analytical interface for the LotoIA platform.

## Sessions

- Criar Jogos
- Resultados Passados
- Historical Intelligence
- Analytics Intelligence
- ML Intelligence
- ML Governance
- Observability
- Reports Engine
- Meus Testes
- Relatórios

## Intelligence Layers

The dashboard now organizes the platform into layered intelligence sections:

- statistical generation;
- historical intelligence;
- analytics intelligence;
- ML intelligence;
- model governance;
- observability;
- report governance.

## Analytics

The visual analytics layer includes:

- frequency charts;
- delay charts;
- sum distribution;
- odd/even balance;
- recurrence tables;
- heatmaps;
- historical trend views.

## ML

The ML section presents:

- score ML;
- reranking outputs;
- feature weights;
- walk-forward validation;
- temporal safety summary;
- experiment snapshot references.

## Observability

The observability section presents:

- runtime logs;
- audit trail;
- health metrics;
- recent events;
- failure tracking;
- operational summaries.

## Reports

The reports section provides:

- PDF exports;
- CSV exports;
- JSON payloads;
- snapshots;
- artifact download controls.

## Layout Philosophy

The dashboard keeps a premium institutional style:

- clean;
- analytical;
- corporate;
- modular;
- readable;
- stable on Streamlit Cloud.
